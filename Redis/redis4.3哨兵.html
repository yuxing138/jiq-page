# 什么是脑裂

1、master0断网，slave转master1 //新master数据非最新
2、master0连网，出现2个master //称为脑裂
3、master0降级slave，offset更新 //数据丢失

# 选举的流程

1、master超时或宕机，则重新选举
2、选谁？master候选者：优先级最高（priority最小）、offset最高（复制数据最多）、runid最小（最先连接master）
3、选票过半，则为master
